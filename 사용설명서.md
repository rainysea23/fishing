# 지도호 낚시 예약 현황 모니터링 시스템 사용설명서

---

## 개요

지도호 낚시 예약 페이지를 1시간마다 자동 크롤링하여 달력 형식으로 보여주고,
주말·공휴일에 빈자리 변경이 생기면 텔레그램으로 즉시 알림을 보내는 시스템입니다.

## 동작 방식

### 자동 실행 주기
- **1시간마다** GitHub Actions가 자동 실행 (PC 꺼도 동작)
- 크롤링 → 데이터 비교 → 페이지 업데이트 → 변경 시 텔레그램 알림

### 텔레그램 알림 조건
주말(토·일) 또는 공휴일 날짜에 아래 변경이 생길 때만 알림 전송:

| 상황 | 알림 내용 |
|------|-----------|
| 마감 → 빈자리 생길 때 | 🆕 빈자리 생겼습니다! N명 |
| 취소로 빈자리 증가 | 📈 N명 → M명으로 증가 |
| 새 날짜 예약 오픈 | 📅 예약 오픈! N명 |

### 달력 페이지 색상 의미

| 색상 | 의미 |
|------|------|
| 초록색 | 예약 가능 (숫자 = 남은 자리 수) |
| 빨간색 | 예약 마감 |
| 노란색 테두리 | 오늘 |
| 회색 | 예약 데이터 없음 |

## 관리 방법

### 일시정지 (자동 실행 멈추기)
1. https://github.com/rainysea23/fishing/actions/workflows/update.yml 접속
2. 우측 `...` 버튼 클릭
3. **Disable workflow** 클릭

### 재시작
1. 같은 페이지에서 **Enable workflow** 클릭

### 수동 실행 (즉시 업데이트)
1. https://github.com/rainysea23/fishing/actions/workflows/update.yml 접속
2. 우측 **Run workflow** 버튼 클릭
3. 초록색 **Run workflow** 클릭

### 오류 확인
1. https://github.com/rainysea23/fishing/actions 접속
2. 빨간색(실패) 항목 클릭 → 로그 확인

## 문제 해결

### 텔레그램 알림이 안 올 때
1. https://github.com/rainysea23/fishing/settings/secrets/actions 에서 Secrets 확인
2. `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` 정확히 입력됐는지 확인
3. 봇 채팅창에서 `/start` 한번 눌렀는지 확인 (@Sangikryu_bot)

### 달력 페이지가 안 열릴 때
1. https://github.com/rainysea23/fishing/settings/pages 확인
2. Source: `Deploy from a branch`, Branch: `main`, Folder: `/ (root)` 설정 확인

### Actions가 계속 실패할 때
- https://github.com/rainysea23/fishing/actions 에서 오류 로그 확인
- 낚시 사이트 서버가 일시적으로 느릴 수 있음 (자동 재시도됨)
